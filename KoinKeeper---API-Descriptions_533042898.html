Dominic CURTIS : KoinKeeper - API Descriptions

Created by Dominic CURTIS, last modified on 14.03.2023

  1. POST /api/users/register - Registers a new user with the system. Requires a JSON payload containing the user's name, email, and password.

  2. POST /api/users/login - Logs in a user to the system. Requires a JSON payload containing the user's email and password.

  3. GET /api/users/{userId} - Retrieves information about a specific user. Requires the user's ID in the URL path.

  4. PUT /api/users/{userId} - Updates information about a specific user. Requires the user's ID in the URL path and a JSON payload containing the updated user information.

  5. DELETE /api/users/{userId} - Deletes a specific user from the system. Requires the user's ID in the URL path.

  6. POST /api/wallets - Creates a new wallet for a user. Requires a JSON payload containing the user's ID and the wallet type (e.g. Bitcoin, Ethereum, Litecoin).

  7. GET /api/wallets/{walletId} - Retrieves information about a specific wallet. Requires the wallet's ID in the URL path.

  8. PUT /api/wallets/{walletId} - Updates information about a specific wallet. Requires the wallet's ID in the URL path and a JSON payload containing the updated wallet information.

  9. DELETE /api/wallets/{walletId} - Deletes a specific wallet from the system. Requires the wallet's ID in the URL path.

  10. POST /api/transactions - Creates a new transaction for a user. Requires a JSON payload containing the user's ID, the wallet ID, the transaction amount, and the transaction type (e.g. Buy, Sell, Transfer).

  11. GET /api/transactions/{transactionId} - Retrieves information about a specific transaction. Requires the transaction's ID in the URL path.
